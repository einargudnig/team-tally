## Directory Purposes

**apps/api/src/**
- Purpose: Backend API source code
- Contains: Hono routes, middleware, utilities
- Key files: `index.ts` (server), `routes/users.ts` (CRUD)
- Subdirectories: `routes/`, `lib/`, `middleware/`, `types/`

**apps/api/src/routes/**
- Purpose: API route handlers
- Contains: Hono route modules
- Key files: `users.ts` (user CRUD), `index.ts` (root routes)

**apps/api/src/lib/**
- Purpose: Shared utilities for API
- Contains: `auth.ts` (better-auth config), `db.ts` (placeholder), `env.ts` (environment)

**apps/web/src/**
- Purpose: React frontend source
- Contains: Components, API client, styles
- Key files: `App.tsx`, `api.ts`, `main.tsx`

**packages/shared/**
- Purpose: Shared code across all apps
- Contains: TypeScript types, utility functions
- Key files: `index.ts` exports `User`, `UserResponse`, `UsersResponse`, `formatDate()`

## Key File Locations

**Entry Points:**
- `apps/api/src/index.ts` - API server entry
- `apps/web/src/main.tsx` - Web app entry

**Core Logic:**
- `apps/api/src/routes/users.ts` - User CRUD operations
- `apps/api/src/lib/auth.ts` - Authentication setup
- `apps/web/src/api.ts` - Type-safe API client

**Testing:**
- No test files exist yet
- Pattern specified: `*.test.ts` with Bun test

**Documentation:**
- `CLAUDE.md` - Bun development guidelines
- `apps/api/IMPLEMENTATION_PLAN.md` - Feature roadmap

## Naming Conventions

**Files:**
- kebab-case for utility files: `auth.ts`, `db.ts`, `env.ts`
- PascalCase for React components: `App.tsx`
- index.ts for module entry points

**Directories:**
- Lowercase singular for categories: `lib/`, `middleware/`, `types/`
- Lowercase plural for collections: `routes/`, `apps/`, `packages/`

**Special Patterns:**
- `index.ts` - Directory/module entry point
- `.env` - Environment variables (Bun auto-loads)
- `*.config.ts` - Tool configuration

## Where to Add New Code

**New API Route:**
- Implementation: `apps/api/src/routes/{resource}.ts`
- Mount in: `apps/api/src/index.ts` via `app.route()`
- Types: `apps/api/src/types/index.ts` or `packages/shared/index.ts`

**New React Component:**
- Implementation: `apps/web/src/components/{Name}.tsx`
- Styles: `apps/web/src/{name}.css` or co-located

**New Shared Type:**
- Definition: `packages/shared/index.ts`
- Export and use in both API and Web

**New Middleware:**
- Implementation: `apps/api/src/middleware/{name}.ts`
- Apply in: `apps/api/src/index.ts` via `app.use()`

**Utilities:**
- API-specific: `apps/api/src/lib/`
- Shared: `packages/shared/`

## Special Directories

**apps/api/dist/**
- Purpose: Compiled API output
- Source: Generated by `bun build`
- Committed: No (in .gitignore)

**apps/web/dist/**
- Purpose: Production web build
- Source: Generated by `vite build`
- Committed: No (in .gitignore)

**node_modules/**
- Purpose: Dependencies
- Source: `bun install`
- Committed: No (in .gitignore)
